O vulgo "Pythônico"

Por que escrever código pythonico?

  • Manutenebilidade
    • Codigo não pythônico é estranho quando analisado por alguém experiente
  • Menor curva de aprendizado
    • Codigo uniforme entre diversos projetos diminui a barreira de entrada
  • Aumenta suas chances de receber contribuições e de ter contribuiçoes aceitas em projetos importantes
    • Código não idiomático não costuma ser aceito na maioria dos projetos mais proeminentes

Espaços 1

Espaço em branco normalmente é importante em qualquer linguagem para melhorar a visualização do código, mas em python é essencial. A sintaxe da linguagem é controlada por espaços, aqui a coisa é seria e um transtorno obcessivo compulsivo é visto com bons olhos.

Convenções fortes:

  • 4 espaços
  • Se preferir 2, 3, 5 espaços, ok, mas seja consistente e não misture
  • Tabs = Não... inconsistente.
  • Tabs + espaços = O inferno tem um lugarzinho reservado para você, certamente
  • Uma linha em branco separando metodos de classes
  • Duas linhas em branco separando funções e classes em um mesmo módulo

Espaços 2

  • Espaços depois de "," em dicionários, listas, tuplas e listas de argumentos e depois de ":" em dicionários
  • Espaços em volta de atribuições e comparações(exceto em listas de argumentos).
  • Sem espaços em docstrings

Ruim:

def bola_quadrada(chave,valor=0):
    """ Retorna um dicionário e uma lista... """
    d={chave:valor}
    l=[chave,valor]
    return d,l

Bom:

def bola_quadrada(chave, valor=0):
    """Retorna um dicionário e uma lista..."""
    d = {chave: valor}
    l = [chave, valor]
    return d, l

Unpacking

Trocando valores em linguagens mais baixo nível:

temp = a
a = b
b = temp

Em Python:

>>> b, a = a, b

O que aconteceu?

  • Uma tupla foi criada no lado direito
  • Uma tupla foi criada no lado esquerdo
  • A tupla da direita foi desempacotada (unpacking) na da esquerda

    ...

    ...

defaultdict

Ao invés de pre-construir um dicionário ou usar o get usando o segundo parametro opcional default você pode usar o defaultdict

Ruim:

>>> c = {}
>>> c.get("gosta_mais", "batata")
'batata'
>>> c.get("o_que_prefere", "batata")
'batata'

Bom:

>>> from collections import defaultdict
>>> c = defaultdict(lambda: "batata")
>>> c['gosta_mais']
'batata'
>>> c['prefere']
'batata'

http://docs.python.org/2/library/collections.html#collections.defaultdict

Named Tuples

Tuplas nomeadas servem para dar significado a posição de um item numa tupla e fazem que o codigo fique mais legível e auto-documentado. Elas funcionam de maneira igual as tuplas.

>>> Point = namedtuple('Point', ['x', 'y'])
>>> p = Point(11, y=22)     # instancia com argumento posicional ou keyword arg
>>> p[0] + p[1]             # indexavel como a tupla normal
33
>>> x, y = p                # unpack como uma tupla normal
>>> x, y
(11, 22)
>>> p.x + p.y               # campos acessaveis pelo nome
33
>>> p                       # __repr__ legivel com estilo nome=valor
Point(x=11, y=22)

http://docs.python.org/2/library/collections.html#collections.namedtuples

Recomendações

O uso de list comprehensions para construção de listas é pythônico e seu uso encorajado sempre que possivel(mas tenha bom senso).

Ruim:

>>> youre_doing_it_wrong = []
>>> for i in range(10):
...     youre_doing_it_wrong.append("%d elem" % i)

Bom:

>>> result = ["%d elem" % i for i in range(10)]

Passou do ponto:

>>> result = [(i, ("%d %d elem" %(i,j))) for i in range(10) for j in range(10)]

Evite nesting nos list comprehensions. Existem casos válidos, mas normalmente causam a perda de legibilidade.

Filtrando

Somente os pares:

Ruim:

>>> youre_doing_it_wrong = []
>>> for i in range(10):
...     if i % 2 == 0:
...         youre_doing_it_wrong.append("%d elem" % i)

Bom:

>>> result = ["%d elem" % i for i in range(10) if i % 2 == 0]

Passou do ponto:

>>> result = ["%d elem" % i for i in range(10) if i % 2 == 0 or i % 5 == 0 or (i == 10 and True==False)]

Normalmente, mais que dois filtros a coisa começa a ficar feia, melhor fazer da maneira tradicional.

Set & Dict Comprehensions

A partir do python 2.7+ e 3+ a linguagem ganhou os dict e set comprehensions.

Python < 2.7:

>>> d = dict((key, value) for (key, value) in sequence)

Novo:

>>> d = {key: value for (key, value) in sequence}

Python < 2.7:

>>> s = set([v for v in 'ABCDABCD' if v not in 'CB'])
>>> s
set(['A', 'D'])

Novo:

>>> s = {v for v in 'ABCDABCD' if v not in 'CB'}
>>> s
{'A', 'D'}

Generators

As vezes voce não precisa usar uma lista inteira, só os passos intermediários bastam. Com generators você consegue fazer isso:

Definindo um gerador que representa todos numeros inteiros positivos

def pos_int():
    i = 0
    while True:
        yield i
        i += 1
>>> ints = pos_int()
>>> ints.next()
0
>>> ints.next()
1
>>> ints.next()
2

Generators 2

Outro exemplo

def walkfiles(startdir, pattern=None):
    """Gerador de paths completos para todos os arquivos que começem
    de `startdir`. Opcionalmente filtra arquivos fora do padrao `pattern`
    """
    for dir, dirlist, filelist in os.walk(startdir):
        for fname in filelist:
            if pattern and not fnmatch.fnmatch(fname, pattern):
                continue
            yield os.path.join(dir, fname)

Zip Tricks

Zip é uma funçãozinha bem útil

>>> lista = ["Charlinho", "queria", "estudar"]
>>> indices = range(3)
>>> zip(indices, lista)
[(0, 'Charlinho'), (1, 'queria'), (2, 'estudar')]

>>> for (elem, i) in zip(lista, indices):
...    print "elem {} indice {}".format(elem, i)
elem Charlinho indice 0
elem queria indice 1
elem estudar indice 2

Criando dicionários

>>> d = dict(zip(indices, lista))
>>> d
{0: "Charlinho", 1: "queria", 2: "estudar"}
